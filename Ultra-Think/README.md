# Ultra Think - アプリアイデア管理プロジェクト

## ⚠️ 重要：プロジェクトルール

### このプロジェクトで行うこと ✅
1. **アプリアイデアの収集と整理**
2. **アイデアの詳細仕様検討**
3. **制作スケジュール管理**
4. **技術要件の洗い出し**
5. **優先順位付け**

### このプロジェクトで行わないこと ❌
1. **実際のアプリ制作・コーディング**
2. **プロトタイプの実装**
3. **テストコードの作成**
4. **デプロイ作業**

> **注意**: 制作が決定したアプリは、必ず**別プロジェクト**として新規作成し、そこで開発を行うこと

---

## プロジェクト概要

Ultra Thinkは、アプリアイデアを管理し、制作スケジュールを立てるための企画専用プロジェクトです。

## ディレクトリ構造

```
Ultra-Think/
├── README.md                    # このファイル（プロジェクトルール）
├── app-ideas/                   # 各アプリアイデアの詳細仕様
│   ├── 01_鼻歌作曲メーカー.md
│   ├── 02_水平思考ゲーム.md
│   ├── 03_最期の砂時計.md    # 🚧 制作中
│   ├── 04_スリメイト.md
│   └── ...
├── schedules/                   # 制作スケジュール管理
│   └── project-timeline.md
└── ideas-list.md               # アイデア一覧（全31個）
```

## アプリステータス

### 🚧 制作中
- **最期の砂時計** - 別プロジェクトで開発進行中

### ✅ 詳細検討済み
- **鼻歌作曲メーカー** - iOS先行、音楽初心者向け
- **水平思考ゲーム** - ウミガメのスープ型謎解き

### 📝 アイデア段階
- **スリメイト** - ダイエット仲間チャットアプリ
- その他27個のアイデア

## 新しいアプリを制作する際のワークフロー

1. `app-ideas/` フォルダ内の詳細仕様を確認
2. 制作スケジュールを `schedules/` で管理
3. **新規プロジェクトを作成**（例：`001-HumComposer`）
4. 新規プロジェクトで実際の開発を開始

## 貢献ガイドライン

- アイデアの追加は `ideas-list.md` に記載
- 詳細検討したアイデアは `app-ideas/` に個別ファイルを作成
- コードの実装は絶対に行わない
